# Updater signing keys (desktop)

## Required CI secrets

Set the following repository or environment secrets in CI for release builds:

- `TAURI_SIGNING_PRIVATE_KEY`: contents of the private key file generated by Tauri signer
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: password used when generating the private key

## Public key location

The updater public key is committed in `src-tauri/tauri.conf.json` under:

- `plugins.updater.pubkey`

## Private key handling

- Never commit the private key file to git.
- Keep the private key in secure storage (for example `~/.clickshield/keys/updater.key`).
- Restrict filesystem permissions on the key file to the current user.

## Key rotation

1. Generate a new keypair with `npx tauri signer generate --ci --write-keys <secure-path> --password "<new-password>"`.
2. Replace `plugins.updater.pubkey` in `src-tauri/tauri.conf.json` with the new public key.
3. Update CI secrets: `TAURI_SIGNING_PRIVATE_KEY` and `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`.
4. Publish a new signed release artifact and updater metadata using the new key.
5. Verify update flow from an installed older app to ensure signature validation succeeds.

## Local build test

Use your local key and password in a shell before building:

```bash
export TAURI_SIGNING_PRIVATE_KEY="$(cat ~/.clickshield/keys/updater.key)"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD="<your-password>"
npm run tauri:build -- --bundles app
```
